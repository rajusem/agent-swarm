# Swarmer — Phase 1: Foundation

**Date:** 2026-04-14
**Status:** Implemented

## Goal

Bootstrap a runnable FastAPI application with authentication, encrypted field storage, and base UI templates. No business features yet — just a solid foundation every subsequent phase builds on.

## Stack

| Concern | Choice |
|---------|--------|
| Web framework | FastAPI + Jinja2 |
| Partials / interactivity | HTMX (CDN) + Bootstrap 5 (CDN) |
| Database | SQLite via `aiosqlite` + SQLAlchemy 2.x async |
| Field encryption | `cryptography` Fernet (field-level) |
| Password hashing | `argon2-cffi` |
| Config | `pydantic-settings` (.env + env vars) |
| Kubernetes client | `kubernetes` Python package |

## `swarmer/auth.py`

```python
import argon2

_ph = argon2.PasswordHasher()

def load_hash(hash_file: Path) -> str:
    return hash_file.read_text().strip()

def verify_password(plain: str, stored_hash: str) -> bool:
    try:
        return _ph.verify(stored_hash, plain)
    except (argon2.exceptions.VerifyMismatchError, argon2.exceptions.VerificationError):
        return False

def hash_password(plain: str) -> str:
    return _ph.hash(plain)
```

## `swarmer/crypto.py`

```python
import base64, hashlib
from pathlib import Path
from cryptography.fernet import Fernet

_fernet: Fernet | None = None
_FALLBACK = "SWARMER_NOT_CONFIGURED_RUN_MAKE_SETUP_AUTH"

def _read_hash_bytes(hash_file: Path) -> bytes:
    try:
        return hash_file.read_bytes()
    except FileNotFoundError:
        return _FALLBACK.encode()

def init_crypto(hash_file: Path) -> None:
    """Derive Fernet key: SHA-256(b"fernet:" + hash_file_bytes) → base64url."""
    global _fernet
    raw = _read_hash_bytes(hash_file)
    key_bytes = hashlib.sha256(b"fernet:" + raw).digest()
    _fernet = Fernet(base64.urlsafe_b64encode(key_bytes))

def derive_session_secret(hash_file: Path) -> str:
    """Derive Starlette session secret: SHA-256(b"session:" + hash_file_bytes).
    Safe to call at import time — falls back to a constant if hash file missing."""
    raw = _read_hash_bytes(hash_file)
    return hashlib.sha256(b"session:" + raw).hexdigest()

def encrypt(plaintext: str) -> str: ...
def decrypt(ciphertext: str) -> str: ...
```

**Key design decision:** `derive_session_secret()` is called at module import time when `SessionMiddleware` is constructed. The hash file may not exist yet (before `make setup-auth` runs). Fallback to a deterministic constant prevents a crash; once `setup-auth` runs and the app restarts, real keys are derived.

## `swarmer/database.py`

```python
from sqlalchemy.ext.asyncio import AsyncSession, async_sessionmaker, create_async_engine
from sqlalchemy.orm import DeclarativeBase

class Base(DeclarativeBase):
    pass

_engine = None
_AsyncSessionLocal: async_sessionmaker | None = None

def init_db(database_url: str) -> None:
    global _engine, _AsyncSessionLocal
    _engine = create_async_engine(database_url, echo=False)
    _AsyncSessionLocal = async_sessionmaker(_engine, expire_on_commit=False)

async def create_tables() -> None:
    import swarmer.models  # registers all model tables on Base.metadata
    async with _engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)

async def get_db() -> AsyncSession:
    async with _AsyncSessionLocal() as session:
        yield session
```

No Alembic. Schema is managed by `create_all` at startup. When a column is added to a model, run `make db-reset` to drop and recreate the database.

## `swarmer/deps.py`

```python
from fastapi import Request

class NotAuthenticated(Exception):
    """Raised when session cookie is missing or invalid."""

def require_auth(request: Request) -> None:
    if not request.session.get("authenticated"):
        raise NotAuthenticated()
```

## `swarmer/main.py`

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI, Request
from fastapi.responses import RedirectResponse
from starlette.middleware.sessions import SessionMiddleware

from swarmer import k8s
from swarmer.config import settings
from swarmer.crypto import derive_session_secret, init_crypto
from swarmer.database import create_tables, init_db
from swarmer.deps import NotAuthenticated

@asynccontextmanager
async def lifespan(app: FastAPI):
    init_crypto(settings.auth_hash_file)
    init_db(settings.database_url)
    await create_tables()
    k8s.init_k8s(settings.k8s_in_cluster)
    yield

app = FastAPI(title="Swarmer", lifespan=lifespan)
app.add_middleware(
    SessionMiddleware,
    secret_key=derive_session_secret(settings.auth_hash_file),
    session_cookie="swarmer_session",
    same_site="lax",
    https_only=False,
)

@app.exception_handler(NotAuthenticated)
async def not_authenticated_handler(request: Request, exc: NotAuthenticated):
    return RedirectResponse(url="/login", status_code=302)

# Routers included: auth, workspaces, secrets, sessions, tui_ws
```

## `scripts/setup_auth.py`

```python
#!/usr/bin/env python3
import getpass, sys
from pathlib import Path
from argon2 import PasswordHasher

pw = getpass.getpass("Password: ")
pw2 = getpass.getpass("Confirm password: ")
if pw != pw2: sys.exit("Passwords do not match.")
if not pw: sys.exit("Password cannot be empty.")

Path("auth").mkdir(exist_ok=True)
Path("data").mkdir(exist_ok=True)
Path("auth/password.hash").write_text(PasswordHasher().hash(pw))
print("Hash written to auth/password.hash")
```

**Why a separate script?** The `make setup-auth` target originally tried `python3 -c "try: ..."` inline, but multi-line `try/except` cannot be expressed in a single-line `-c` string with `\n` escapes. Moving to `scripts/setup_auth.py` resolved the `SyntaxError`.

## Resetting the Database

When model columns change (e.g. adding a new field), SQLAlchemy's `create_all` does not run ALTER TABLE. Drop and recreate:

```bash
make db-reset   # deletes data/swarmer.db
# then restart the app — create_all recreates all tables
```

For `make dev`: just restart `uvicorn`.
For kind: rebuild and redeploy (`make kind-deploy`), or `kubectl rollout restart deployment/swarmer -n swarmer`.

## Flash Messages (`swarmer/flash.py`)

Flash messages are stored in `request.state` (per-request, not persisted across redirects). They are set with `flash(request, message, category)` and rendered in `base.html` as Bootstrap alerts. Categories map to Bootstrap contextual colours: `success`, `warning`, `danger`, `info`.

## Auth Routes (`swarmer/routers/auth.py`)

| Method | Path | Description |
|--------|------|-------------|
| GET | `/login` | Render `login.html` |
| POST | `/login` | Verify password; set `session["authenticated"] = True`; redirect to `/` |
| POST | `/logout` | Clear session; redirect to `/login` |
