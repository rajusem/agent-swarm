# Swarmer — Phase 2: Workspace Management

**Date:** 2026-04-14
**Status:** Implemented
**Depends on:** Phase 1 (Foundation)

## Goal

Full workspace CRUD. Workspaces map 1:1 to Kubernetes namespaces. The detail page shows inline session status. Deletion requires typing the workspace name to confirm.

## Database Schema

```sql
CREATE TABLE workspaces (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    display_name TEXT    NOT NULL,
    namespace    TEXT    NOT NULL UNIQUE,   -- DNS label, max 63 chars, immutable
    description  TEXT    NOT NULL DEFAULT '',
    created_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

## Model (`swarmer/models/workspace.py`)

```python
class Workspace(Base):
    __tablename__ = "workspaces"
    id:           Mapped[int]      = mapped_column(primary_key=True)
    display_name: Mapped[str]      = mapped_column(String(255))
    namespace:    Mapped[str]      = mapped_column(String(63), unique=True)
    description:  Mapped[str]      = mapped_column(Text, default="")
    created_at:   Mapped[datetime] = mapped_column(DateTime, server_default=func.now())
    updated_at:   Mapped[datetime] = mapped_column(DateTime, server_default=func.now(),
                                                   onupdate=func.now())
    sessions: Mapped[list["Session"]] = relationship(back_populates="workspace",
                                                     cascade="all, delete-orphan")
```

## Namespace Derivation

```python
def _derive_namespace(display_name: str) -> str:
    slug = display_name.lower()
    slug = re.sub(r"[^a-z0-9]+", "-", slug)
    return slug.strip("-")[:63]
```

Rules (RFC 1123 DNS label):
- Lowercase
- Replace runs of non-`[a-z0-9]` with a single `-`
- Strip leading/trailing hyphens
- Truncate to 63 characters
- Unique — `IntegrityError` on collision surfaces as a form error
- **Immutable after creation**

### Detail Route

Loads sessions with `selectinload(Session.github_pat)`, passes them to `detail.html` for the inline sessions table.

```python
@router.get("/workspaces/{ws_id}")
async def workspace_detail(ws_id, request, db):
    ws = await db.get(Workspace, ws_id)
    ns_status = k8s.get_namespace_status(ws.namespace)
    sessions = (await db.execute(
        select(Session).where(Session.workspace_id == ws_id)
        .options(selectinload(Session.github_pat))
        .order_by(Session.name)
    )).scalars().all()
    return templates.TemplateResponse("workspaces/detail.html",
        {"request": request, "ws": ws, "ns_status": ns_status, "sessions": sessions})
```

## Delete Confirmation Flow (HTMX)

1. User clicks "Delete Workspace" → `hx-get="/workspaces/{id}/delete"` swaps the button with `_delete_confirm.html`
2. Partial: "Type **{display_name}** to confirm" + text input + red Delete button
3. POST `/workspaces/{id}/delete` receives `confirm_name` form field
4. Mismatch → re-render partial with inline error (no redirect, no DB change)
5. Match → `k8s.delete_namespace()` first; if non-404 error, show error and abort; otherwise `db.delete(ws)` + redirect to `/workspaces` with flash

## Templates

### `workspaces/list.html`
- Bootstrap table: Display Name (link to detail), Namespace (`<code>`), K8s Status badge, Actions
- Empty state card with "Create your first workspace" CTA

### `workspaces/new.html`
- `display_name` input with `hx-get="/workspaces/preview-namespace" hx-trigger="input delay:300ms"` updating a live namespace preview `<code>` span
- `description` textarea
- Error alert div for server-side validation failures (duplicate namespace)

### `workspaces/detail.html`
- Heading: `display_name` + namespace `<code>` badge + K8s namespace status badge (Active / Unknown)
- Inline sessions table: Name, Mode badge (Prompt/Server/TUI), Phase badge, PAT name, Persist flag, Open button
- "New Session" button
- Top-right: Secrets link + Edit link

### `workspaces/edit.html`
- Form: display_name (editable) + description (editable) + namespace (disabled, immutable)
- Below `<hr>`: Danger Zone card with HTMX delete-confirmation flow

### `workspaces/_delete_confirm.html`
- Red alert box
- Input: `placeholder="{{ ws.display_name }}"`, `name="confirm_name"`
- Submit button POSTs to `/workspaces/{id}/delete`
- Error message shown inline if name mismatch

## Error Handling

| Scenario | Behaviour |
|----------|-----------|
| Duplicate namespace | Form error: "A workspace with namespace `xyz` already exists." |
| K8s namespace create fails | Flash warning; DB record still created |
| K8s namespace delete fails (non-404) | Inline error in delete confirm partial; DB untouched |
| Confirm name mismatch | Inline error in partial; no delete |
| Workspace not found | Redirect to `/workspaces` |
