# Swarmer — Phase 4: Session Management

**Date:** 2026-04-14
**Status:** Implemented
**Depends on:** Phase 3 (Secrets)

## Goal

Sessions are the operational unit of work: a Kubernetes PVC-backed `/workspace` folder paired with an opencode container. The dashboard manages the full lifecycle (create → launch → monitor → stop → delete), git repository tracking, and three launch modes including a browser TUI via xterm.js + WebSocket + pty.

## Database Schema

```sql
CREATE TABLE sessions (
    id                INTEGER PRIMARY KEY AUTOINCREMENT,
    workspace_id      INTEGER NOT NULL REFERENCES workspaces(id) ON DELETE CASCADE,
    github_pat_id     INTEGER REFERENCES github_pats(id) ON DELETE SET NULL,
    name              TEXT NOT NULL,
    mode              TEXT NOT NULL DEFAULT 'prompt',  -- 'prompt' | 'server' | 'tui'
    persist           INTEGER NOT NULL DEFAULT 0,   -- BOOL: PVC survives session stop
    resume            INTEGER NOT NULL DEFAULT 0,   -- BOOL: --continue on re-launch
    instruction_prompt TEXT NOT NULL DEFAULT '',    -- used in prompt mode only
    pod_name          TEXT,                         -- K8s pod name when active
    pvc_name          TEXT,                         -- stable: "session-{id}"
    last_output       TEXT NOT NULL DEFAULT '',     -- last 50 lines of pod logs
    phase             TEXT NOT NULL DEFAULT 'idle', -- idle|pending|running|succeeded|failed|stopped
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE (workspace_id, name)
);

CREATE TABLE session_repos (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id  INTEGER NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
    repo_url    TEXT NOT NULL,
    branch      TEXT NOT NULL DEFAULT 'main',
    local_path  TEXT NOT NULL,   -- relative path inside /workspace
    UNIQUE (session_id, local_path)
);
```

**Schema migration note:** The original `interactive_mode` (BOOL) column was replaced with `mode` (TEXT). Run `make db-reset` when this change is applied — SQLAlchemy `create_all` does not run ALTER TABLE.

## Session Modes

| Mode | Pod command | Restart policy | Service | Browser terminal |
|------|-------------|---------------|---------|-----------------|
| `prompt` | `opencode run --model <model> [--continue] "<prompt>"` | Never | No | No |
| `server` | `opencode serve --hostname 0.0.0.0` | Always | ClusterIP on port 4096 | No |
| `tui` | `sleep infinity` | Always | No | Yes (xterm.js + kubectl exec) |

```python
class Session(Base):
    __tablename__ = "sessions"
    __table_args__ = (UniqueConstraint("workspace_id", "name"),)

    id:                Mapped[int]      = mapped_column(primary_key=True)
    workspace_id:      Mapped[int]      = mapped_column(ForeignKey("workspaces.id"))
    github_pat_id:     Mapped[int|None] = mapped_column(ForeignKey("github_pats.id"), nullable=True)
    name:              Mapped[str]      = mapped_column(String(255))
    mode:              Mapped[str]      = mapped_column(String(16), default="prompt", server_default="prompt")
    persist:           Mapped[bool]     = mapped_column(Boolean, default=False)
    resume:            Mapped[bool]     = mapped_column(Boolean, default=False)
    instruction_prompt: Mapped[str]     = mapped_column(Text, default="")
    pod_name:          Mapped[str|None] = mapped_column(String(255), nullable=True)
    pvc_name:          Mapped[str|None] = mapped_column(String(255), nullable=True)
    last_output:       Mapped[str]      = mapped_column(Text, default="")
    phase:             Mapped[str]      = mapped_column(String(32), default="idle")
    repos:             Mapped[list["SessionRepo"]] = relationship(
                           back_populates="session", cascade="all, delete-orphan")

    @property
    def interactive_mode(self) -> bool:
        """True for modes that keep the pod running persistently (tui or server)."""
        return self.mode in ("tui", "server")

    @property
    def is_active(self) -> bool:
        return self.phase in ("pending", "running")
```

### Mode-specific container args

```python
if session.mode == "server":
    args = ["opencode", "serve", "--hostname", "0.0.0.0"]
    restart_policy = "Always"
    ports = [client.V1ContainerPort(container_port=4096, name="opencode")]
    stdin, tty = False, False

elif session.mode == "tui":
    args = ["sleep", "infinity"]   # pod stays alive; kubectl exec opens opencode
    restart_policy = "Always"
    ports = []
    stdin, tty = True, True        # allows kubectl exec -it

else:  # prompt
    args = ["opencode", "run", "--model", _DEFAULT_MODEL]
    if session.resume:
        args.append("--continue")
    if session.instruction_prompt:
        args.append(session.instruction_prompt)
    restart_policy = "Never"
    ports = []
    stdin, tty = False, False
```

### Init container (git clone)

When the session has repos, an `alpine/git:latest` init container runs before the main container:

```sh
# Per repo:
[ -d /workspace/{local_path}/.git ] || \
  git clone {repo_url} --branch {branch} /workspace/{local_path}
```

Git credential setup (HTTPS auth via stored credentials file):
```sh
if [ -n "${GITHUB_PAT}" ]; then
    git config --global credential.helper store
    echo "https://${GITHUB_USERNAME}:${GITHUB_PAT}@github.com" > /root/.git-credentials
fi
```

The init container receives `GITHUB_PAT` and `GITHUB_USERNAME` env vars from the PAT K8s Secret (same pattern as the main container). **Lessons learned:** The original implementation used `GIT_ASKPASS` pointing to a non-existent script. The credential.helper store approach is more reliable.

## Session Lifecycle

```
idle → launch → pending → running → succeeded/failed  (prompt mode)
idle → launch → pending → running                     (server/tui mode, stays running)
running → stop → stopped
stopped → launch → pending → ...                      (re-launch)
idle/stopped → delete → (gone)
```

On **stop**:
1. `k8s.delete_pod(session.pod_name, ws.namespace)`
2. If `session.mode == "server"`: `k8s.delete_service(f"session-{id}-svc", namespace)`
3. If `not session.persist and session.pvc_name`: `k8s_sess.delete_session_pvc(…)`
4. `session.phase = "stopped"`, `session.pod_name = None`

On **delete** (must be idle or stopped):
1. `k8s_sess.delete_session_pvc(ws.namespace, session.pvc_name)`
2. `db.delete(session)` — cascades to `session_repos`

## HTMX Status Polling

`_status_badge.html` — polled every 5 s while `phase in ('pending', 'running')`:

```html
<div id="session-status"
  {% if session.phase in ('pending', 'running') %}
  hx-get="/workspaces/{{ ws.id }}/sessions/{{ session.id }}/status"
  hx-trigger="every 5s"
  hx-swap="outerHTML"
  {% endif %}>

  <!-- Phase badge: ● Active | ⏳ Pending — ErrImagePull | ✗ Failed — OOMKilled | … -->
  <!-- Pod name in monospace -->

  {# Auto-reload for TUI mode: terminal requires a page reload to get a fresh tui_token #}
  {% if session.mode == 'tui' and session.phase == 'running' %}
  <script>
    if (!document.getElementById('terminal')) {
      window.location.reload();
    }
  </script>
  {% endif %}
</div>
```

The `/status` route:
1. `k8s.get_pod_status(pod_name, namespace)` → `(our_phase, detail)`
2. `k8s.get_pod_logs(pod_name, namespace, tail_lines=50)` → updates `session.last_output`
3. `session.phase = our_phase` → commit
4. Returns `_status_badge.html` partial

Status detail priority (most specific first):
1. Init-container waiting reason (`PodInitializing`, `ErrImagePull`)
2. Main container waiting reason (`ContainerCreating`, `ImagePullBackOff`, `CrashLoopBackOff`)
3. Main container terminated reason (`OOMKilled`, `Completed`, `Error`)
4. Raw K8s phase string as fallback

Polling stops automatically when phase reaches a terminal state — the partial is rendered without the `hx-trigger` attribute.

## TUI WebSocket (`swarmer/routers/tui_ws.py`)

### Authentication

The detail route generates a one-time UUID token stored in `request.session["tui_tokens"]` (a list). The token is embedded in the page HTML. The WebSocket handler receives the token as the first message and removes it from the session list. Invalid/missing tokens close the connection with code 4001.

### The PTY Problem

`kubectl exec -it` requires the subprocess to see a real TTY on its stdin/stdout. Running it via `asyncio.create_subprocess_exec` with `PIPE` for stdio produces:

```
Unable to use a TTY - input is not a terminal or the right kind of file
```

**Fix:** Use `pty.openpty()` to create a master/slave pseudo-terminal pair. Pass the slave fd as the subprocess's stdin/stdout/stderr. kubectl sees a real TTY and correctly allocates one inside the pod.

### Implementation

```python
import asyncio, fcntl, os, pty

@router.websocket("/ws/{ws_id}/sessions/{sid}/tui")
async def session_tui(websocket, ws_id, sid):
    await websocket.accept()

    # 1. Receive and validate one-time token
    token = await asyncio.wait_for(websocket.receive_text(), timeout=10)
    tui_tokens = websocket.session.get("tui_tokens", [])
    if token not in tui_tokens:
        await websocket.close(code=4001, reason="Invalid token"); return
    tui_tokens.remove(token)

    # 2. Load session from DB; verify mode=="tui" and phase=="running"

    # 3. Open PTY pair
    master_fd, slave_fd = pty.openpty()
    proc = await asyncio.create_subprocess_exec(
        "kubectl", "exec", "-it", pod_name, "-n", namespace, "--", "opencode",
        stdin=slave_fd, stdout=slave_fd, stderr=slave_fd,
    )
    os.close(slave_fd)  # parent doesn't need the slave end

    # 4. Make master non-blocking for asyncio add_reader
    flags = fcntl.fcntl(master_fd, fcntl.F_GETFL)
    fcntl.fcntl(master_fd, fcntl.F_SETFL, flags | os.O_NONBLOCK)

    # 5. asyncio Queue bridges the pty reader callback to the async read_loop
    loop = asyncio.get_event_loop()
    read_queue: asyncio.Queue[bytes | None] = asyncio.Queue()

    def _pty_readable():
        try:
            data = os.read(master_fd, 4096)
            loop.call_soon_threadsafe(read_queue.put_nowait, data or None)
        except OSError:  # EIO when slave side closes (process exited)
            loop.call_soon_threadsafe(read_queue.put_nowait, None)

    loop.add_reader(master_fd, _pty_readable)

    # 6. Bi-directional proxy
    async def read_loop():    # pty master → browser
        while chunk := await read_queue.get():
            await websocket.send_bytes(chunk)

    async def write_loop():   # browser → pty master
        while data := await websocket.receive_bytes():
            os.write(master_fd, data)

    # 7. Wait for either loop to finish, cancel the other
    read_task = asyncio.create_task(read_loop())
    write_task = asyncio.create_task(write_loop())
    done, pending = await asyncio.wait([read_task, write_task],
                                       return_when=asyncio.FIRST_COMPLETED)
    for t in pending: t.cancel()

    # 8. Cleanup
    loop.remove_reader(master_fd)
    os.close(master_fd)
    proc.terminate()
```

## `imagePullPolicy: IfNotPresent`

**Critical for locally loaded kind images.** Kubernetes defaults to `Always` for `:latest` tags, which always tries to pull from the registry. When using locally loaded images (via `kind load image-archive`), the pod must have `imagePullPolicy: IfNotPresent` to use the local copy.

Applied in two places:
1. `k8s/swarmer/deployment.yaml` — for the swarmer app pod itself
2. `k8s_session.py` `build_session_pod()` — for session pods:
   ```python
   container = client.V1Container(
       name="opencode",
       image=image,
       image_pull_policy="IfNotPresent",
       ...
   )
   ```

## Resetting the Database

The `mode` column (TEXT, `prompt|server|tui`) replaced the original `interactive_mode` (BOOL) column. SQLAlchemy's `create_all` does not run `ALTER TABLE`, so existing databases must be reset:

```bash
make db-reset
# Restart the app — create_all creates fresh tables with the new schema
```

For the kind-deployed version:
```bash
kubectl exec -n swarmer deployment/swarmer -- rm -f /data/swarmer.db
kubectl rollout restart deployment/swarmer -n swarmer
```
