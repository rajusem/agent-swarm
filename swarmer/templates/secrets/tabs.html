Configure a GitHub App to replace per-session PATs with short-lived Installation Access Tokens minted server-side by Swarmer. The raw private key never enters the sandbox.

Sessions authenticate using the GitHub App first; a session-assigned PAT acts as fallback when no App is configured. For long-running TUI and server-mode sessions, Swarmer automatically refreshes the token before it expires. See docs/GITHUB_APP_SETUP.md for setup steps and required GitHub App permissions.
